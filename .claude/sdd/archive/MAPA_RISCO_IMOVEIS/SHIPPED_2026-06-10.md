## Lições Aprendidas

### Processo
- **Ordem de dependências importa**: schemas → sources → dimensions → calculator → models → routes. Respeitar essa ordem eliminou erros de importação circular.
- **Fixtures de teste = prática padrão, não dados fake**: confusão inicial sobre fixtures. Clarificar desde o início que fixtures de teste são apenas para unit tests isolados.

### Técnico
- **`httpx.Response` requer `request=` para `raise_for_status()`**: em versões recentes do httpx, criar um `Response` sem o objeto `Request` correspondente faz `raise_for_status()` levantar `RuntimeError`. Sempre construir mocks com `httpx.Response(..., request=httpx.Request("GET", url))`.
- **`unicodedata.normalize("NFD")` é essencial para strings portuguesas**: o CNJ retorna "Execução Fiscal" com acento; busca por substring sem normalização falha silenciosamente. Normalizar para ASCII antes de qualquer comparação de texto em fontes externas brasileiras.
- **`except Exception: return []` swallows everything**: resiliente para produção, mas torna o debugging em testes opaco. Padrão correto: log o erro antes de retornar lista vazia, mesmo que seja `logger.debug`.
- **PostGIS já habilitado**: não exigiu trabalho adicional na migration — só criar a tabela com `AddGeometryColumn` e o índice GIST.
- **`score_partial=True` é o comportamento correto**: quando qualquer fonte externa está indisponível, o score deve ser calculado com as dimensões disponíveis e sinalizado como parcial — não bloquear o cálculo.

### Ferramentas
- **Ruff `--fix`** resolveu todos os 6 erros de lint automaticamente (import ordering + unused import).
- **WeasyPrint + Jinja2**: combinação limpa para geração de PDF — o template HTML/CSS fica versionado junto ao código.
